# sei09 Bash起動構成の思想

この `.bashrc` は、sei09環境における「再現性・構成美・操作効率」を追求した設計です。  
Copilotとの連携や公開運用を前提に、以下の思想で構成されています。

---

## 🧠 起動メッセージの目的

- `clear` によって前回のログを消去し、**構成確認メッセージが明瞭に表示される**
- `echo` によって、**作業者への挨拶と現在地の明示**
- `find . -maxdepth 1 -type d` によって、**ファイルを除いたディレクトリ構成のみを表示**
  - 視認性を保ちつつ、**構成の骨格を確認する儀式**として残している

---

## 🔧 Git操作の簡略化

- `g`, `ga`, `gc`, `gs`, `gp` などの alias により、**Git操作を最小入力で実行可能**
- 命名は直感的かつ一貫性を持たせている

---

## 📁 ディレクトリ移動の命名思想

- `cdlab`, `cdco`, `cdrepo` は、**「cd + 意味」形式で違和感を解消**
- 単語だけで移動する alias（例：`lab`）は避け、**コマンドらしさと思想の両立を図っている**

---

## 📝 ファイル編集ショートカット

- `ignore`：公開用 `.gitignore` を即編集
- `bootstrap`：Copilot連携用 README を即編集

---

## 💡 Git Bashの注意点

- Ctrl+V は効かないため、**右クリック または Shift+Insert でペースト**
- `.bashrc` は `/c/Users/ユーザー名/` に配置する必要がある

---

## 🧩 補足

この構成は、**sei09流の運用美学と再現性の追求**を反映したものです。  
公開にあたっては、**コメントとドキュメントで思想を明示し、他者にも伝わる設計**を目指しています。
