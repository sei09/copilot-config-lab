githubのWEBサイトとGitbushで操作しながらSSH鍵を再度設定した。

## 公開アドレス yahooメールのほう。Gmailと混同しやすいので注意

公開するアドレスは
sei09 / <sei0921@yahoo.com>

git config --global user.name "sei09"
git config --global user.email "sei0921@yahoo.com"

git config --global user.name
git config --global user.email
必要なら既存リポジトリのローカル設定も上書きします（その場合は各リポジトリで --global なしで同じコマンド実行）。続けてSSH接続確認（ssh -T git@github.com）まで一緒にやりますか？

切り分け結果は明確です。ローカル側は正常で、GitHub側に鍵が認識されていません。

現在使っている鍵の指紋: SHA256:98uuQQeroWBoqCXlN4wlwbXARZGP+iFLK4WFTQkO1b4
テスト結果: Permission denied (publickey)（= GitHubアカウントに未登録/別鍵登録/別アカウント）
次をそのまま実施してください（最短）:

公開鍵をクリップボードへ: Get-Content $HOME/.ssh/id_ed25519.pub | Set-Clipboard
GitHubの Settings > SSH and GPG keys で新規追加
Key type は Authentication Key を選択
追加後に再テスト: ssh -T -i $HOME/.ssh/id_ed25519 -o IdentitiesOnly=yes git@github.com
再テストの出力を貼ってください。成功メッセージが出るまでこちらで詰めます。

一般的には「用途ごとに分けて、見て分かる名前」にします。おすすめは次です。

既定鍵として使うなら id_ed25519（最も一般的）
サービス別に分けるなら id_ed25519_github / id_ed25519_gitlab
端末別に分けるなら id_ed25519_github_desktop-win など
公開鍵コメント（-C）は mail + 端末名 + 日付 が管理しやすい
例: sei0921@yahoo.com win11-main 2026-02

GitHub側にその公開鍵が正しく登録されていない です。
まずこの順で進めてください（最短ルート）。

現在鍵の指紋を確認
ssh-keygen -lf ~/.ssh/id_ed25519.pub
（表示は SHA256:a3Ae4zB97d58BH5Dfi5eUcMo/X+ML2q61RDsA9twVns のはず）
GitHubの Settings > SSH and GPG keys で、同じ鍵をAuthentication Keyとして登録
既存の同種鍵が紛らわしければ一旦削除して、今の .pub を再登録
cat ~/.ssh/id_ed25519.pub
再テスト
ssh -T -i ~/.ssh/id_ed25519 -o IdentitiesOnly=yes git@github.com
まだ失敗する場合は、衝突回避で新規鍵に切り替えます（確実）。必要なら次で一気に案内します。
実行結果を貼ってください。

sei09@SUNAMO MINGW64 ~
$ ssh -T -i ~/.ssh/id_ed25519 -o IdentitiesOnly=yes git@github.com
Hi sei09! You've successfully authenticated, but GitHub does not provide shell access.

ssh -T -i ~/.ssh/id*ed25519 -o IdentitiesOnly=yes git@github.com
